# RustChat 阶段成果路线图

本文档结合 `docs/summary/` 阶段记录和当前 `dev` 分支的 Git 提交历史，按时间轴整理 RustChat 的阶段成果、演进路线和交付重点。

## 1. 整理口径

- 时间线以当前 `dev` 分支 `git log --date=short --reverse` 为准。
- 阶段成果以 `docs/summary/README.md` 及各 phase summary 为主要依据。
- Phase 编号来自项目开发任务拆分；时间线按真实提交日期排列，所以部分体验增强阶段会出现在 Phase 6 之后。
- `docs/summary` 中曾出现的旧提交引用，如果不在当前主线日志中，本文不作为主线时间点展开。

## 2. 总体路线

```text
项目启动
  -> 后端基础设施
  -> 实时私聊主链路
  -> 会话列表与历史消息
  -> 前端工程与浏览器联调
  -> 群聊与成员一致性
  -> 真实数据 UI 收口
  -> WebSocket 重连与发送体验
  -> 文件上传稳定化
  -> 文档结构整理
  -> 媒体消息、群主成员管理与会话实时刷新
```

这条路线体现了 RustChat 的核心演进方式：先建立可运行工程，再打通私聊最小闭环，随后扩展会话体验、浏览器可用性、群聊、多媒体消息和演示验收能力。

## 4. 阶段成果分组

### 4.1 工程与基础设施

覆盖阶段：Phase 0、Phase 1。

主要成果：

- 后端 Rust 工程初始化。
- 统一配置、日志、错误处理和响应结构。
- PostgreSQL 连接、migration 和核心表设计。
- 用户注册、登录、JWT 鉴权和认证中间件。
- API 文档和 Postman collection 起步。

路线意义：

- 先解决“项目能启动、能连库、能鉴权、能调试”的基础问题。
- 为后续所有业务模块提供统一工程能力。

### 4.2 私聊主链路

覆盖阶段：Phase 2、Phase 3。

主要成果：

- WebSocket 连接基础。
- 私聊会话创建。
- 文本消息发送、落库、推送和发送确认。
- 会话列表、最近消息、未读数。
- 历史消息分页。
- 真实数据库端到端测试。
- 用户搜索、会话已读、私聊唯一性。

路线意义：

- 形成 RustChat 第一个完整聊天闭环。
- 把“能发消息”扩展为“能查历史、能看列表、能验真实库”。

### 4.3 前端可用与浏览器联调

覆盖阶段：Phase 0、Phase 4、Phase 6。

主要成果：

- Vue 3 + Vite 前端工程。
- API 封装、Pinia 状态管理和基础页面。
- 聊天 UI、会话切换和消息展示。
- 浏览器 HTTP 联调修复。
- 刷新后恢复当前会话。
- 前端真实数据展示和样式收口。

路线意义：

- 将后端接口能力转化为用户可操作的浏览器体验。
- 用真实浏览器场景暴露 CORS、状态恢复和数据展示问题。

### 4.4 群聊与成员一致性

覆盖阶段：Phase 5、Phase 5.5.1/2、Phase 5.5.6-8。

主要成果：

- 群聊创建、添加成员和退出群聊。
- 群消息广播。
- 群成员列表。
- 退出群聊后的本地状态清理。
- 群主移除普通成员。
- 成员变化后通过 `conversation_updated` 触发会话实时刷新。

路线意义：

- 将系统从一对一聊天扩展到多人会话。
- 补强成员变化后的前端、后端和实时状态一致性。

### 4.5 实时体验与消息可靠性

覆盖阶段：Phase 5.5.3、Phase 5.5.4。

主要成果：

- WebSocket 自动重连。
- 在线、连接中、重连中、离线状态展示。
- 重连后重新拉取会话和当前会话历史。
- 本地乐观消息。
- `queued / sending / sent / failed` 状态。
- pending 队列、重连后自动补发和失败重试。

路线意义：

- 从“在线时能实时聊天”升级为“异常网络下也能给用户明确反馈”。
- 降低断线、重连、发送失败带来的体验割裂。

### 4.6 文件与媒体消息

覆盖阶段：Phase 5.6、Phase 5.5.6-8。

主要成果：

- 文件分片上传。
- 上传 owner 校验、chunk 校验、SHA256 校验。
- 文件入库、消息入库和会话最近消息事务一致。
- 文件下载和错误提示。
- 图片/GIF 内联预览。
- 本地自定义表情。
- 语音录制、上传和播放器展示。

路线意义：

- 将消息类型从文本扩展到文件和媒体。
- 保持媒体消息复用文件上传链路，避免新增过多后端复杂度。

## 5. 路线总结

RustChat 的阶段路线可以概括为四个层次：

1. 基础设施层：先完成工程、数据库、配置、日志、认证和接口文档。
2. 核心业务层：打通私聊、消息、会话列表、历史消息和真实数据库测试。
3. 用户体验层：补齐前端 UI、浏览器联调、刷新恢复、重连和消息发送状态。
4. 扩展能力层：加入群聊、成员管理、文件上传、图片/GIF、语音和会话实时刷新。

当前项目已经从“后端练手项目”演进为具备前后端、数据库、WebSocket、文件媒体消息、群聊管理和验收文档的轻量级 IM 系统。后续路线建议继续围绕稳定性、验收材料和部署说明收口，不再盲目扩大到音视频通话、复杂权限、分布式部署或商业级高并发能力。
