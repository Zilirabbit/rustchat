## Phase 5.5：WebSocket 体验、消息发送体验与媒体增强

- `1b0eb03`：WebSocket 重连状态
- `46117c7`：扩展 WebSocket 体验任务

本阶段做了什么：

- 支持断线后自动重连。
- 重连成功后重新拉取会话列表和当前会话历史消息。
- 支持消息发送本地乐观展示。

- 支持发送失败后手动重试。
- 增加 GIF / 表情面板。
- 支持图片 / GIF 内联预览和保存为本地表情。
- 支持语音录制、上传和播放。
- 增加群主移除普通成员能力。
- 增加 `conversation_updated` WebSocket 事件，通知相关用户刷新会话。

阶段产出：

- WebSocket 自动重连机制。
- 重连后的数据补偿机制。
- 消息发送状态和 pending 队列。
- GIF / 表情能力。
- 语音消息能力。
- 群主成员管理能力。
- 会话实时刷新事件。

阶段验收：

- 断线后前端显示重连状态。
- 后端恢复后前端能重新连接并刷新数据。
- 离线或重连中发送的消息能进入 pending 队列。
- 重连后 pending 消息可自动补发。
- 图片/GIF 可预览，语音消息可播放。
- owner 可以移除普通群成员。
- 成员变化后相关用户会话列表能实时刷新。

## Phase 5.6：文件上传稳定 v1

代表提交：

- `b14d040`：新增文件上传功能
- `f813a60`：更新 gitignore
- `1e46189`：合并文件上传分支
- `7866465`：细化重连布局和验证说明
- `4a0898d`：文件上传 v1 稳定化

本阶段做了什么：

- 增加文件上传功能。
- 支持上传初始化、分片上传、完成上传和下载。
- `upload_id` 绑定上传发起人。
- 校验文件元数据、chunk 顺序、实际大小和 SHA256。
- 文件完成入库改为事务处理，保证文件记录、消息记录和会话最近消息一致。
- 失败时清理已移动到 final 的文件。
- final 文件名加入 UUID，降低重名冲突。
- 下载接口生成安全 `Content-Disposition`。
- 前端支持上传进度、防重复选择、空文件和超过 100MB 文件拦截。
- 记录 Web Crypto digest 不可用的已知问题。

阶段产出：

- `file` 模块和文件上传 API。
- `files` 表和文件元信息存储。
- 本地 `UPLOAD_DIR` 文件存储。
- 文件消息入库与 WebSocket 推送。
- 文件上传稳定性 summary。
- 文件上传已知问题记录。

阶段验收：

- 用户可以上传小文件。
- 上传完成后会产生附件消息。
- 会话另一端可以实时收到附件消息。
- 成员可以下载文件。
- 空文件、超大文件、hash 不匹配等异常场景能被拒绝。
- 后端测试、前端构建和 diff 检查通过。

## 总结

按 Phase 拆解后，RustChat 的建设路径可以理解为：

1. Phase 0 解决项目启动和前后端工程基础。
2. Phase 1 解决后端基础设施、数据库、用户和鉴权。
3. Phase 2 打通私聊、消息、会话列表和历史消息主链路。
4. Phase 3 补齐真实数据库测试、用户搜索、已读状态和私聊唯一性。
5. Phase 4 将接口能力接入浏览器前端并完成本地联调。
6. Phase 5 扩展群聊和成员一致性。
7. Phase 5.5 强化实时连接、消息发送体验、媒体消息和群主控制。
8. Phase 5.6 稳定文件上传和文件消息链路。
9. Phase 6 收口真实数据 UI 和设计资产。
10. 文档补强阶段将成果沉淀为可查、可演示、可复盘的项目资料。

整体来看，项目从基础工程逐步演进到完整聊天闭环，再扩展到群聊、文件、媒体消息和演示验收能力，路线清晰且每个阶段都有相对独立的交付成果。
