# Summary 记录规范

`docs/summary` 用于记录阶段性开发结果、关键变更和验证结果。

## 命名规则

建议使用下面的文件名格式：

```text
v<版本号>-<阶段>-<主题>.md
```

例如：

```text
v0.1.0-phase1-project-init.md
v0.2.0-phase1-common.md
```
