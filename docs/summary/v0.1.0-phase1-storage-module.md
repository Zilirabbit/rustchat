## 关键实现说明

### 1. storage 不再只是一个裸 `PgPool`

当前 `storage` 模块提供：

- `Storage`：统一管理数据库连接与仓储上下文
- `db.rs`：负责建连、migration 执行、数据库 ping
- `repository.rs`：提供后续业务仓储统一复用的上下文与 trait

这样后续 `user / session / message` 模块可以直接围绕 `repo.rs` 扩展，而不需要重复定义数据库访问入口。

### 2. migration 在启动时自动执行

当配置了 `DATABASE_URL` 后，应用启动时会：

1. 建立连接池
2. 自动执行 `backend/migrations/`
3. 将可用的 `Storage` 注入 `AppState`

这保证本地初始化和后续开发环境启动流程更稳定。

### 3. 连接池参数已纳入配置管理

新增环境变量：

- `DATABASE_MAX_CONNECTIONS`
- `DATABASE_MIN_CONNECTIONS`
- `DATABASE_ACQUIRE_TIMEOUT_SECS`

当前默认值分别为 `10 / 1 / 5`，能满足当前阶段的本地开发和基础联调。
