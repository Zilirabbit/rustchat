## 本次完成内容

- 新增 `POST /api/sessions/group` 创建群聊接口
- 新增 `POST /api/sessions/{session_id}/members` 添加群成员接口
- 新增 `DELETE /api/sessions/{session_id}/members/me` 退出群聊接口
- 群聊创建者自动成为 `owner`，初始成员自动去重并写入 `session_members`
- 添加群成员时校验当前用户必须是群主，重复添加会返回已有成员关系
- 退出群聊时校验当前用户必须是群成员；群主退出后会自动提升最早加入的成员为新群主
- 最后一名成员退出后自动删除该群聊会话
- WebSocket `send_message` 从私聊单接收者推送扩展为会话成员广播，群消息会推送给所有在线成员（排除发送者）
- 更新 `docs/api/http-api.md` 与 `docs/product/task.md`

## 关键实现说明

### 1. 群聊继续复用 session 模块

本次没有新增单独的 group 模块，而是沿用现有 `session` 领域：

- `sessions.session_type = 'group'`
- 群名称写入 `sessions.name`
- 群成员与角色写入 `session_members`

这样可以继续复用会话列表、历史消息、已读状态和成员权限校验。

### 2. 群消息广播继续走 message + connection

`message service` 现在会基于 `session_members` 取得当前会话的其他成员列表：

- 发送者必须属于该会话
- 消息先落库并更新 `sessions.last_message_id / last_message_at`
- WebSocket handler 对 `recipient_user_ids` 逐个推送 `receive_message`
- 离线成员不会阻塞发送方确认

因此私聊和群聊共享同一个 `send_message` 协议。

### 3. 权限边界

当前群聊权限先收敛在基础规则：

- 创建者为 `owner`
- 只有 `owner` 可以添加群成员
- 普通成员可以退出群聊
- 非成员不可添加成员、退出群聊、发送消息或查询历史消息

后续如需管理员、多群主或转让群主，可以在 `session_members.role` 上继续扩展。

### 2. 手动 HTTP 验证

准备三个用户并分别登录：

1. `POST /api/register` 注册 Alice、Bob、Carol
2. `POST /api/login` 获取三人的 token
3. Alice 调用 `POST /api/sessions/group`

示例请求：

```json
{
  "name": "integration team",
  "member_user_ids": [2]
}
```

预期：

- 返回 `message = "group session created"`
- `data.session_type = "group"`
- `data.created_by` 是 Alice 的用户 ID
- `data.member_user_ids` 同时包含 Alice 和 Bob

Alice 添加 Carol：

```text
POST /api/sessions/<group_id>/members
```

请求体：

```json
{
  "user_id": 3
}
```

预期：

- 返回 `message = "group member ready"`
- `data.user_id` 是 Carol 的用户 ID
- `data.added = true`

权限反向验证：

- Bob 调用同一个添加成员接口，预期返回 `403`
- 未加入群聊的用户发送群消息或查询历史消息，预期返回 `403`

### 3. 手动 WebSocket 验证

分别用 Alice、Bob、Carol 的 token 建立三个 WebSocket 连接：

```text
GET /ws?token=<jwt>
```

每个连接成功后，预期先收到：

```json
{
  "type": "connected"
}
```

Alice 向群聊发送消息：

```json
{
  "type": "send_message",
  "session_id": 22,
  "content": "hello group"
}
```

预期：

- Alice 收到 `message_sent`
- Bob 收到 `receive_message`
- Carol 收到 `receive_message`
- Bob 和 Carol 收到的 `message` 与 Alice 的 `message_sent.message` 一致

### 4. 会话列表与退出验证

Bob 查询会话列表：

```text
GET /api/conversations
```

预期：

- 能看到刚创建的群聊
- `session_type = "group"`
- `session_name` 为群名称
- `last_message` 为 Alice 刚发送的群消息
- `unread_count = 1`

Carol 退出群聊：

```text
DELETE /api/sessions/<group_id>/members/me
```

预期：

- 返回 `message = "group session left"`
- `data.user_id` 是 Carol 的用户 ID
- `data.left = true`

Carol 再次查询 `GET /api/conversations`：

- 不应再看到该群聊
