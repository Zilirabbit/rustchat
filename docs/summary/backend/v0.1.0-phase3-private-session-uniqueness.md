## 关键实现说明

### 1. 迁移会清空现有私聊

本次按已确认要求执行：

```sql
UPDATE sessions
SET last_message_id = NULL,
    last_message_at = NULL
WHERE session_type = 'private';

DELETE FROM sessions
WHERE session_type = 'private';
```

先清空最近消息指针是为了避免已有私聊消息被 `sessions.last_message_id` 外键保护而无法删除。随后由于现有外键使用级联删除，对应私聊消息、私聊成员关系、私聊已读状态会一起清理。群聊 session 和群聊相关数据不会被删除。

### 2. 私聊唯一性由数据库兜底

新表 `private_session_pairs` 保存排序后的用户对：

```text
user_low_id < user_high_id
UNIQUE (user_low_id, user_high_id)
```

这样无论请求方向是 Alice -> Bob 还是 Bob -> Alice，都会落到同一个唯一键。

### 3. 并发创建保持接口幂等

服务层仍先查询已有私聊。若多个请求同时通过查询并进入创建，数据库唯一约束会只允许一个事务写入 pair。其他事务捕获冲突后回查已有会话，并按原接口返回：

```text
created = false
```

HTTP 请求和响应字段保持不变。
