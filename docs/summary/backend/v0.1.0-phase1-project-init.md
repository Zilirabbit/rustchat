## 关键实现说明

### 1. 启动解耦数据库

为满足 3.1 “项目可成功启动、健康检查可访问”的目标，服务启动不再强依赖 PostgreSQL。

当前行为：

- 若配置了 `DATABASE_URL`，启动时初始化数据库连接池
- 若未配置 `DATABASE_URL`，服务仍可启动并提供基础接口

这样可以将“项目初始化”与“数据库模块接入”分开，符合任务拆分：

- `3.1` 先保证服务可运行
- `3.3` 再完善数据库初始化和迁移体系
