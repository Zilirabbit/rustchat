## 本次完成内容

本次补齐 Phase 3 前端体验收尾项：用户刷新 `/chat` 后，不再只恢复登录态，而是恢复刷新前正在查看的会话。

### 前端状态恢复

- 新增当前会话缓存 key：`rustchat.chat.activeSessionId`
- 用户点击会话或创建私聊成功后，将当前 `activeSessionId` 写入 `localStorage`
- 页面刷新进入 `/chat` 后：
  - 先恢复 token / user 登录态
  - 再拉取 `GET /api/conversations`
  - 如果缓存的会话仍属于当前用户，自动选中该会话
  - 自动拉取 `GET /api/messages?session_id=<id>&limit=20`
  - 自动调用 `POST /api/sessions/<id>/read`
  - 使用已恢复 token 重新建立 WebSocket 连接
- 退出登录时清理当前会话缓存，避免切换用户后误恢复旧会话

### 设计取舍

本次只持久化当前会话 ID，不持久化消息列表。

原因：

- 消息以服务端数据库为准，刷新后重新拉历史消息更可靠
- 避免浏览器缓存与服务端历史消息产生不一致
- 保持前端状态恢复逻辑简单，适合当前 Phase 3 演示范围

如果缓存的会话 ID 已不在当前用户会话列表中，前端会清理该缓存并保持未选择会话状态。

## 浏览器验收标准

1. 登录用户 A，打开与用户 B 的私聊会话。
2. 发送一条消息，确认当前消息列表正常显示。
3. 刷新页面。
4. 仍然停留在 `/chat`。
5. 左侧刷新前打开的会话仍处于选中状态。
6. 右侧自动显示该会话历史消息。
7. WebSocket 自动重连，顶部显示“实时已连接”。
8. 刷新后继续发送新消息正常。
9. 点击退出后重新登录其他用户，不会恢复上一个用户打开的会话。
