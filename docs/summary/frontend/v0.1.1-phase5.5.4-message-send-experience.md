# RustChat Summary

## 版本信息

- 版本：`v0.1.1`
- 阶段：`phase5.5.4`
- 主题：消息发送体验
- 日期：`2026-05-12`

## 已实现能力

本次补齐消息发送体验：前端支持本地乐观消息、发送状态展示、失败重试、断线 pending 队列，以及 WebSocket 重连后的自动补发。

### 发送状态

- 发送消息时立即插入本地消息，减少等待后端 ack 的空白感
- 本地消息支持 `queued / sending / sent / failed` 状态
- 消息列表展示 `等待重连 / 发送中 / 已发送 / 发送失败`
- 失败消息展示 `重试` 入口
- 发送超时后自动标记为失败，避免长期停留在发送中

### pending 队列与补发

- WebSocket 未连接、断线或重连中发送的消息进入当前页面内存 pending 队列
- WebSocket 重连成功后，先执行现有会话与历史消息补偿，再自动补发 pending 消息
- 补发成功后用后端真实消息替换本地乐观消息，避免重复展示
- 补发失败后保留失败状态，并允许用户手动重试

### WebSocket 协议增强

- 前端 `send_message` 增加可选 `client_message_id`
- 后端 `message_sent` 回传同一个 `client_message_id`
- 后端发送消息失败时，`error` 事件回传同一个 `client_message_id`
- 不带 `client_message_id` 的旧客户端请求继续兼容

## 暂不实现能力

- pending 队列不写入 `localStorage`，刷新页面后不恢复
- 不做跨 tab pending 同步；多窗口协调仍属于 `5.5.3.4`
- 不做消息草稿、附件、撤回、编辑和已读回执

## 浏览器端验证方案

### 准备

1. 启动后端服务。
2. 启动前端 dev server。
3. 准备两个用户 A、B，分别在两个浏览器窗口登录。

### 在线发送

1. A 打开与 B 的私聊或共同群聊。
2. A 发送一条消息。
3. 确认 A 侧消息状态从 `发送中` 变为 `已发送`。
4. 确认 B 侧收到同一条消息。

### 断线 pending 与自动补发

1. A 打开一个会话并保持页面不刷新。
2. 停止后端服务，或让 A 的浏览器网络离线。
3. A 继续发送消息。
4. 确认消息留在列表中，并显示 `等待重连`。
5. 恢复后端服务或网络。
6. 等待 WebSocket 重连成功。
7. 确认 pending 消息自动补发，并变为 `已发送`。

### 失败与重试

1. 制造失败场景，例如向已失效或当前用户已离开的会话发送消息。
2. 确认消息显示 `发送失败`。
3. 点击 `重试`。
4. 若连接可用且后端允许发送，确认消息重新进入 `发送中` 并最终变为 `已发送`；若仍失败，确认保留失败状态。

### 连续相同内容

1. 连续发送多条内容完全相同的消息。
2. 确认每条消息状态独立更新，不会互相错配。
