## 本次完成内容

本次将前端聊天页与后端已有群聊能力对齐，完成创建群聊、添加成员、退出群聊、群消息收发和刷新恢复的基础闭环。

### 前端群聊能力

- 新增侧栏内联群聊面板
- 支持填写群聊名称、搜索用户、多选初始成员并创建群聊
- 创建成功后刷新会话列表，并自动打开新群聊
- 选中群聊后支持搜索并添加成员
- 选中群聊后支持当前用户退出群聊
- 退出当前群聊后清理 `activeSessionId`、该群聊历史消息缓存，并刷新会话列表

### 接口与状态管理

- 前端新增 `POST /api/sessions/group`、`POST /api/sessions/<id>/members`、`DELETE /api/sessions/<id>/members/me` 封装
- 补齐群聊创建、添加成员、退出群聊的 TypeScript response 类型
- 群聊消息复用现有 WebSocket `send_message` 协议
- 实时群消息复用现有消息列表与会话预览更新逻辑
- 页面刷新后复用现有 `activeSessionId` 恢复逻辑重新打开群聊

## 浏览器验收标准

1. A、B、C 三个用户分别注册并登录。
2. A 搜索 B，创建包含 B 的群聊。
3. A 打开群聊发送消息，B 在线时收到 `receive_message`。
4. A 搜索 C 并添加到当前群聊。
5. C 刷新会话列表后看到该群聊。
6. A/B/C 在群聊内互发消息，确认会话预览和未读数更新。
7. B 打开群聊后调用已读接口，确认该群聊 `unread_count = 0`。
8. B 退出群聊，再次查询会话列表时不再看到该群聊。
9. A 或 C 刷新 `/chat`，确认仍可恢复当前群聊和历史消息。
