## 本次完成内容

本次补齐前端 WebSocket 连接体验：连接异常关闭后会自动重连，侧栏连接状态可区分在线、连接中、重连中和离线，重连失败后给出明确提示。

### 前端连接体验

- `connection` store 记录当前 token，用于非主动断开后的自动重连
- 自动重连最多尝试 5 次，退避间隔从 1s 增长到最高 8s
- 主动退出登录或页面卸载时清理 socket、重连 timer 和连接状态
- 重连成功后清空错误提示并重置重试次数
- 消息发送仍只允许在 WebSocket 已连接时进行，避免离线发送造成状态错觉

### 界面反馈

- 连接状态文案支持 `Realtime online`、`Connecting`、`Reconnecting n/5`、`Offline`
- 重连中、在线、离线状态使用不同 pill 样式
- 重连失败后提示用户刷新页面或重新登录

## 验证建议

```bash
cd frontend && npm run build
cd backend && cargo test
```

浏览器验收建议：

1. 登录后进入 `/chat`，确认状态显示为 `Realtime online`。
2. 停止后端服务，确认状态进入 `Reconnecting n/5`。
3. 在重试次数耗尽前恢复后端服务，确认状态回到 `Realtime online`。
4. 刷新 `/chat`，确认登录态、当前会话、历史消息与 WebSocket 连接恢复。
