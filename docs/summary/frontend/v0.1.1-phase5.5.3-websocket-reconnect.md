# RustChat Summary

## 版本信息

- 版本：`v0.1.1`
- 阶段：`phase5.5.3`
- 主题：WebSocket 自动重连与连接状态提示
- 日期：`2026-05-12`

## 已实现能力

本次补齐前端 WebSocket 连接体验：连接异常关闭后会自动重连，侧栏连接状态可区分在线、连接中、重连中和离线，重连失败后给出明确提示。

### 基础连接体验

- `connection` store 记录当前 token，用于非主动断开后的自动重连
- 自动重连最多尝试 5 次，退避间隔从 1s 增长到最高 8s
- 主动退出登录或页面卸载时清理 socket、重连 timer 和连接状态
- 重连成功后清空错误提示并重置重试次数
- 消息发送仍只允许在 WebSocket 已连接时进行，避免离线发送造成状态错觉

### 状态反馈

- 连接状态文案支持 `Realtime online`、`Connecting`、`Reconnecting n/5`、`Offline`
- 重连中、在线、离线状态使用不同 pill 样式
- 重连失败后提示用户刷新页面或重新登录

### 重连后数据补偿

- 重连成功后自动重新拉取 `GET /api/conversations`
- 若当前存在打开的会话，自动重新拉取该会话历史消息
- 若当前存在打开的会话，自动重新调用已读接口
- 当前会话历史消息以服务端返回为准覆盖本地列表，避免重复插入
- 会话列表刷新后，若当前会话仍属于当前用户，则继续保留选中状态
- 若当前会话已不再属于当前用户，则清理选中态、本地历史缓存和本地会话 ID

## 暂不实现能力

- 连接质量增强：最近连接成功时间、最近断开时间、下一次重连倒计时、更细的主动离线/网络异常状态区分
- 心跳与超时检测：前端定期 `ping`、后端 `pong` 超时判断、tab 可见性下的心跳频率调整
- 多窗口连接协调：`BroadcastChannel` 同步登录、登出和连接状态，多 tab 避免重复重连提示
- 消息发送增强：离线 pending 队列、重连后自动补发、发送中/已发送/失败/手动重试状态

## 浏览器端验证方案

### 准备

1. 启动后端服务。
2. 启动前端 dev server。
3. 准备两个用户 A、B，分别在两个浏览器窗口登录。

### 私聊重连补偿

1. A 打开与 B 的私聊会话。
2. 确认 A 侧状态为 `Realtime online`。
3. 停止后端服务。
4. 确认 A 侧状态变为 `Reconnecting n/5`。
5. 恢复后端服务。
6. 确认 A 侧状态回到 `Realtime online`。
7. 确认 A 侧会话列表重新刷新，当前会话仍被选中，历史消息仍正常展示。

### 漏消息补偿

1. A 打开与 B 的私聊。
2. 保持后端服务运行。
3. 在 A 的浏览器 tab 里打开 DevTools，把 Network 设置为 Offline，或只让 A 所在浏览器/机器断网。
4. 确认 A 显示 Reconnecting n/5。
5. B 继续在线，并发送一条新消息。
6. 恢复 A 的网络，最好在 5 次重连耗尽前恢复。
7. A 重连成功后，确认：

- 会话列表最近消息更新。
- 当前会话历史消息包含 B 断线期间发送的新消息。
- 如果当前会话被标记已读，未读数归零。

### 未读状态补偿

1. A 不打开某个会话时，让 B 发送消息，确认未读数增加。
2. A 打开该会话并断线。
3. 恢复连接后确认当前会话会重新标记已读，未读数归零。

### 群聊补偿

1. A、B、C 在同一群聊。
2. A 打开群聊后断线。
3. B 或 C 发送群消息。
4. A 重连成功后确认群聊历史消息和会话预览更新。

### 刷新恢复

1. A 打开一个私聊或群聊。
2. 刷新 `/chat`。
3. 确认登录态恢复、当前会话恢复、历史消息加载、WebSocket 状态回到 `Realtime online`。

### 失败路径

1. 停止后端超过 5 次重连周期。
2. 确认状态不再无限重连，并显示重连失败提示。
3. 恢复后端后刷新页面，确认可重新连接。
