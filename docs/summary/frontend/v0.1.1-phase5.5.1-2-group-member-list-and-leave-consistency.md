## 本次完成内容

本次合并完成 `phase5.5.1` 和 `phase5.5.2`：群聊打开后可以看到成员列表，添加成员后会刷新成员缓存；当前用户退出群聊后，会同步清理选中会话、历史消息、成员列表、本地会话 ID，并忽略该群聊后续残留实时消息。

### 后端接口

- 新增 `GET /api/sessions/{session_id}/members`
- 仅当前群成员可查询成员列表
- 返回 `user_id`、`username`、`role`、`joined_at`
- 成员列表按群主优先、加入时间排序
- 补充 service 与 route 单元测试

### 前端体验

- 新增群成员列表 API 封装与 TypeScript 类型
- `ChatView` 打开群聊时加载成员列表
- `GroupChatPanel` 展示成员头像、用户名和角色
- 添加成员成功后刷新成员列表
- 已在群内的用户搜索结果会被禁用，避免重复添加

### 退出一致性

- 退出当前群聊后清空 `activeSessionId`
- 删除该群聊历史消息缓存和成员列表缓存
- 清理 `localStorage` 中保存的当前会话 ID
- 立即从本地会话列表移除该群聊，并重新拉取 `GET /api/conversations`
- 记录已退出会话 ID，忽略该会话后续残留 WebSocket 消息，避免被重新插回会话列表

## BUGFIX

### 1. 创建群聊后无法再添加成员

- 现象：创建群聊并自动打开后，继续搜索用户添加成员时状态不一致，导致无法完成添加或成员列表不刷新。
- 根因：创建群聊后的当前会话、成员缓存和群聊操作状态没有形成完整闭环。
- 修复：创建群聊成功后自动选中新群聊并加载成员列表；添加成员成功后重新拉取成员列表。
- 验证：创建群聊后立即添加第三个用户，确认接口成功、成员列表刷新、已有成员搜索结果被禁用。
