## 问题定位与修复

### 1. 浏览器 CORS 预检失败

联调时先用 `curl` 验证后端接口，`GET /health` 和 `GET /db/ping` 均返回 `200`。

但模拟浏览器预检时：

```text
OPTIONS /api/login
```

最初返回 `405 Method Not Allowed`。

修复方式：

- 新增 `tower-http` 的 `cors` feature 依赖。
- 在 router 外层添加 `CorsLayer`。
- 只允许当前开发环境所需 origin、`GET / POST / OPTIONS` 方法，以及 `Authorization / Content-Type` 请求头。

### 2. 空会话标记已读返回 500

真实 HTTP 流程跑到：

```text
POST /api/sessions/{session_id}/read
```

在空会话场景下返回：

```json
{"code":500,"message":"database error"}
```

定位后确认原因是：

- `sessions.last_message_id` 是可空字段。
- repo 层读取时按非空 `i64` 解码，空会话会触发 `UnexpectedNullError`。

修复方式：

- 将 `last_message_id` 显式按 `Option<i64>` 解码。
- `mark_session_read` SQL 中将 `$3` 显式 cast 为 `BIGINT`，保证 `NULL` 参数写入稳定。
