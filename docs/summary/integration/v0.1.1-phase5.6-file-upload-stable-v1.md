# RustChat Summary

## 版本信息

- 版本：`v0.1.1`
- 阶段：`phase5.6`
- 主题：文件上传稳定 v1
- 日期：`2026-05-13`

## 已实现能力

- 后端补齐文件上传稳定性：`upload_id` 绑定发起人，chunk/complete 校验 owner、文件元数据、chunk 顺序、实际大小和 SHA256。
- 文件完成入库改为事务：写 `files`、写 `messages`、更新会话最近消息在同一事务内完成；事务失败时清理已移动到 final 的文件。
- final 文件名加入 UUID；下载接口安全生成 `Content-Disposition`，并对过期文件返回 not found。
- 启动与后台清理会处理 `UPLOAD_DIR/tmp`、`UPLOAD_DIR/final`，并清理残留临时上传目录。
- 前端统一附件按钮行为，上传中防重复选择，提前拒绝空文件和超过 100MB 文件，上传完成后刷新当前会话与会话列表。
- 下载失败改为页面 toast 提示。

## 暂不实现

- 不做图片内联预览、断点续传、对象存储 / S3、跨进程上传状态持久化。
- v1 继续使用本地 `UPLOAD_DIR`，默认 `./uploads`。

## 已知问题 / 下一阶段调整

- 宿主机通过 `http://192.168.221.131:5173` 访问前端时，部分浏览器不会提供 `crypto.subtle`。
- 当前前端上传会调用 `crypto.subtle.digest` 计算 SHA256，因此可能报 `Cannot read properties of undefined (reading 'digest')`。
- 问题记录：`docs/problems/file_upload_crypto_subtle_digest_issue.md`
- 下一阶段建议保留 Web Crypto 优先路径，并增加纯 JS SHA256 fallback，例如 `@noble/hashes`；同时把原始异常转换成用户可读错误。

## 浏览器交付信息

启动后端：

```bash
cd backend
cargo run
```

启动前端：

```bash
cd frontend
npm run dev -- --host 0.0.0.0
```

访问地址：

- 本机浏览器：`http://127.0.0.1:5173`
- VM / 宿主机浏览器：`http://192.168.221.131:5173`
- 后端健康检查：`http://127.0.0.1:3000/health`
- VM / 宿主机后端健康检查：`http://192.168.221.131:3000/health`

浏览器验收步骤：

1. 准备两个用户 A / B，分别在两个浏览器窗口登录。
2. A 打开与 B 的私聊或共同群聊，上传一个小文件。
3. 确认 A 侧显示进度，完成后出现附件消息。
4. 确认 B 侧实时收到附件消息。
5. A 或 B 点击下载，确认文件名、大小、内容正确。
6. 上传空文件或超过 100MB 文件，确认页面提示错误且不产生消息。
