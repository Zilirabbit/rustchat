## 已实现能力

本阶段补齐聊天体验中的三类增强：GIF / 表情、语音消息、群主成员管理，并把群成员变更后的会话列表刷新接入 WebSocket 实时通知。

### GIF / 表情

- 将原 `Stickers` 与 `GIFs` 入口合并为单一 `GIF` 面板。
- GIF 面板提供默认 GIF / 表情资源。
- 支持从本地上传图片或 GIF 并作为文件消息发送。
- 支持将消息中的图片 / GIF 保存到浏览器本地表情库。
- 保存后的表情可在 GIF 面板中复用。
- 图片 / GIF 消息在消息列表中以内联预览展示。

### 语音消息

- 输入框新增麦克风录音入口。
- 支持开始录音、停止录音和取消录音。
- 录音中展示计时状态。
- 录音结束后生成音频文件，并复用现有文件上传链路发送。
- 前端在缺失 MIME 时按扩展名兜底推断音频类型。
- 消息列表支持 `audio/*` 与 `.webm/.m4a/.mp3/.wav/.ogg/.oga` 文件名兜底识别。
- 语音消息以内联 `<audio controls>` 播放器展示。

### 群主成员管理

- 后端新增 `DELETE /api/sessions/{session_id}/members/{user_id}`。
- 仅群聊 `owner` 可移除普通成员。
- 禁止通过移除成员接口移除群主。
- 禁止 `owner` 通过移除成员接口移除自己，自己退出仍使用 `DELETE /api/sessions/{session_id}/members/me`。
- 前端仅 `owner` 展示添加成员搜索入口。
- 前端仅 `owner` 对普通成员展示 `Remove` 按钮。
- 移除成功后刷新会话列表和成员列表。

### 会话实时刷新

- 后端新增 `conversation_updated` WebSocket 事件。
- 创建群聊后，服务端通知被邀请成员刷新会话状态。
- 添加群成员后，服务端通知新成员刷新会话状态。
- 移除群成员后，服务端通知被移除用户和剩余在线群成员刷新会话状态。
- 前端收到 `conversation_updated` 后重新拉取会话列表；若当前打开的是对应群聊，则同步刷新成员列表。

## 协议与数据约定

- GIF / 表情和语音消息复用文件上传链路，后端仍以 `message_type = "file"` 持久化。
- 图片 / GIF 主要通过 `image/*` MIME 类型识别。
- 语音主要通过 `audio/*` MIME 类型识别，历史数据或浏览器 MIME 缺失时按文件扩展名兜底。
- `conversation_updated` 只传递需要刷新的 `session_id`，具体会话、成员和未读状态仍通过 HTTP API 重新拉取，避免 WebSocket 事件承载过多业务快照。

## 暂不实现能力

- 不做服务端跨设备表情库同步；自定义表情继续存储在浏览器 `localStorage`。
- 不做实时语音通话；当前能力是录制后发送的语音消息。
- 不做多个 owner、owner 转让、管理员角色或复杂群权限。
- 不做端到端媒体转码、缩略图服务或对象存储迁移。
